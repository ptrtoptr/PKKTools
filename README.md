## Необходимые настройки системы

Для использования плагина рекомендуется установить сертификаты Минцифры в системе или в самой QGIS, так как Публичная кадастровая карта НСПД использует данные сертификаты.

Для установки сертификатов исключительно для QGIS, следует открыть настройки и в разделе Аунтификация добавить корневой сертификат.

![Кнопка управления сертификатами](doc/img/Кнопка%20управления%20сертификатами.png "Кнопка управления сертификатами")

![Установка корневого сертификата](doc/img/Установка%20корневого%20сертификата.png "Установка корневого сертификата")

![Импортирование сертификатов](doc/img/Импортирование%20сертификатов.png "Импортирование сертификатов")

Также скорее всего необходимо сменить настройку префикса `User-Agent` со значения по умолчанию `Mozilla/5.0` на, например `Test`. Неизвестно почему, но без смены префикса сайт не отвечает на запросы.

![Настройка префикса User-Agent](doc/img/Настройка%20префикса%20User-Agent.png "Настройка префикса User-Agent")

## Установка плагина

Так как плагин на данный момент не загружен в репозиторий QGIS, плагин устанавливается из архива данного плагина, или копированием папки в корневую папку плагинов QGIS.

## Открытие окна плагина

Основное окно плагина открывается нажатием кнопки "PKKTools" в панели инструментов.

## Основное окно плагина

Интерфейс основного окна плагина состоит из нескольких частей.

### Ввод параметров

Реализованы два вида наборов параметров поиска объектов:

1. Поиск по виду объекта и запросу по кадастровому номеру.

![Параметры по виду объекта](doc/img/Параметры%20вид%20объекта.png "Параметры по виду объекта")

Доступны все виды, доступные на Публичной кадастровой карте НСПД.

![Параметры по виду объекта – выбор вида объекта](doc/img/Параметры%20вид%20объекта%20выбор.png "Параметры по виду объекта – выбор вида объекта")

2. Поиск по точке, входящей в границы объекта.

![Параметры по точке](doc/img/Параметры%20точка.png "Параметры по точке")

Так как на геопортале НСПД слои организованы в древовидной структуре, в плагине слои организованы так же.

![Параметры по точке – первый уровень выбора слоя](doc/img/Параметры%20точка%20первый%20уровень.png "Параметры по точке – первый уровень выбора слоя")

![Параметры по точке – второй уровень выбора слоя](doc/img/Параметры%20точка%20второй%20уровень.png "Параметры по точке – второй уровень выбора слоя")

После выбора слоя необходимо ввести координаты и систему координат точки. Есть две опции на установку точки. Можно ввести координаты и систему координат вручную. Иначе, можно ввести параметры выбором курсором с карты – при выборе с карты автоматически устанавливаются координаты с карты и система координат карты.

![Параметры по точке – установленная точка](doc/img/Параметры%20точка%20установлена.png "Параметры по точке – установленная точка")

![Параметры по точке – установленная точка](doc/img/Параметры%20точка%20установлена.png "Параметры по точке – установленная точка")

### Отображение выбранных параметров

Параметры отображаются в виде древа, где на первом уровне кратко описывается набор параметров, а на втором – сами параметры и их значения. Выделенные выбранные параметры возможно удалить нажатием клавиши Del.

### Операции ввода-вывода выбранных параметров

Операции ввода-вывода выбранных параметров: копирование, сохранение, вставка, загрузка параметров. Формат представления параметров – текстовый формат CSV. Представление параметров: первый столбец – тип набора параметров, остальные – сами параметры. Слои записываются как их идентификаторы, а проекции – как их краткий идентификатор, включающий в себя название организации, например EPSG, и номер проекции. Результат сохранения выбранных параметров приведен ниже.

```csv
object_type,1,77:07:0014006:2675
object_type,1,77:07:0014006:1
point,36048,4172292.03077064,7493516.220316248,EPSG:3857
point,36048,4172409.9202584806,7493413.372549343,EPSG:3857
```

### Настройки загруженного слоя

Указывается настройка на загрузку атрибутов – когда опция выключена, атрибуты не загружаются в итоговый слой. Также указывается выходной файл слоя, как в стандартных алгоритмах фреймворка QGIS Processing. Можно оставить поле пустым, тогда загрузится временный слой, не сохраненный на диск.

В конце окна параметров отображены кнопки загрузки объектов по введенным параметрам, открытия настроек и закрытия окна.

### Загрузка слоя

При нажатии кнопки загрузки начинается процесс загрузки и слой загружается в проект.

![Загруженные объекты на карте](doc/img/МИРЭА%20и%20МГИМО%20скачанные%20на%20карте.png "Загруженные объекты на карте")

## Окно настроек плагина

Окно настроек плагина, отображается отдельно при нажатии кнопки открытия настроек в окне выбора параметров. Настройки перечислены ниже.

1. Настройка загрузки неполных слоёв – при включенной загрузке неполных слоев при ошибке загрузки слоя загружаются слои с успешно загруженными данными на момент ошибки. При отключенной настройке данные слоёв, загруженные не полностью, отвергаются. По умолчанию настройка включена.

2. Настройка количества параллельных запросов – контролирует максимальное количество параллельно выполняемых запросов. При увеличении количества возможно уменьшение общего времени загрузки, однако следует соблюдать осторожность так как большое количество параллельных запро-сов может чрезмерно нагружать сервера НСПД. Значение по умолчанию: 1.

3. Задержка между запросами – после каждого выполненного запроса ожидается заданное количество времени перед выполнением следующего запроса. Следует не понижать значение настройки слишком сильно так как частые запросы могут чрезмерно нагружать сервера НСПД. Значение времени задается в миллисекундах, по умолчанию 1000 мс (1 секунда).

4. Время ожидания запроса – время, после которого запрос отменяется и завершается с ошибкой. Необходимо на случаи, когда сервер не отвечает и необходимо ограничить время ожидания ответа от сервера. Настройка может задаваться, как время ожидания запроса в самом QGIS – оно настраивается внутри параметров ГИС в разделе сети, со значением по умолчанию 60 секунд. Настройка также может устанавливаться для самого плагина отдельно. По умолчанию используется время ожидания, настроенное в QGIS.
